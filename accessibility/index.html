長野県150周年記念事業実行委員会（以下「当実行委員会」といいます）は、長野県150周年記念事業に関する情報を、年齢、性別、障害の有無、利用環境に関係なく、すべての人が等しく利用できるウェブサイトの提供を目指しています。

本サイトは、日本産業規格JIS X 8341-3:2016（高齢者・障害者等配慮設計指針－情報通信における機器，ソフトウェア及びサービス－第3部：ウェブコンテンツ）に対応し、Web Content Accessibility Guidelines (WCAG) 2.1のレベルAA準拠を目指しています。

1. 基本方針

当実行委員会は、以下の基本方針に基づいてアクセシブルなウェブサイトの構築・運営を行います。

🌐 ユニバーサルアクセス

年齢、性別、国籍、身体的制約、技術的制約に関わらず、すべての人が情報にアクセスできるサイトを提供します。

📱 マルチデバイス対応

パソコン、タブレット、スマートフォンなど、さまざまなデバイスで同等の情報とサービスを提供します。

♿ 支援技術対応

スクリーンリーダー、音声ブラウザ、キーボードナビゲーションなどの支援技術との互換性を確保します。

🔄 継続的改善

利用者からのフィードバックを受け、継続的にアクセシビリティの向上に取り組みます。

2. 対象範囲

本アクセシビリティ方針は、以下の範囲に適用されます。

📄 対象ページ

  • 長野県150周年記念事業サイト（https://nagano-150th-a.vercel.app）内のすべてのページ
  • サブドメインを含む関連ページ
  • イベント申込フォーム等の機能ページ
  • PDF等のダウンロード可能ファイル

🚫 対象外

  • 外部サイトへのリンク先
  • YouTube等の外部埋め込みコンテンツ
  • 第三者が提供するウィジェット・プラグイン
  • 緊急時のみ使用される臨時ページ

📅 適用時期

  • 新規作成ページ： 公開時点から適用
  • 既存ページ： 2025年3月末までに準拠
  • 更新ページ： 更新時点で準拠
  • 定期見直し： 年1回の全体チェック

3. 準拠レベル

本サイトは以下の基準への準拠を目指しています。

🎯 主要準拠基準

JIS X 8341-3:2016 レベルAA

日本の工業規格に基づく高齢者・障害者等配慮設計指針

🌍 国際基準

WCAG 2.1 レベルAA

Web Content Accessibility Guidelines 2.1

W3C勧告の国際的なアクセシビリティガイドライン

4つの基本原則

👁️ 知覚可能（Perceivable）

情報とユーザーインターフェースのコンポーネントは、ユーザーが知覚できる方法で提示される必要があります。

  • テキストの代替
  • 音声・映像の代替
  • 適応可能なコンテンツ
  • 判別可能な表示
⌨️ 操作可能（Operable）

ユーザーインターフェースのコンポーネントとナビゲーションは操作可能である必要があります。

  • キーボードアクセス
  • 発作の防止
  • 十分な時間の提供
  • ナビゲーション支援
🤔 理解可能（Understandable）

情報とユーザーインターフェースの操作は理解可能である必要があります。

  • 読みやすさ
  • 予測可能な機能
  • 入力支援
  • エラー修正の支援
🔧 頑健（Robust）

コンテンツは、様々なユーザーエージェントによって確実に解釈される必要があります。

  • 標準準拠
  • 支援技術の互換性
  • 将来性の確保
  • 多様な環境での動作

4. アクセシビリティ確保の取り組み

本サイトでは、以下の取り組みを実施してアクセシビリティを確保しています。

🎨 デザイン・表示

色とコントラスト
  • WCAG AA基準以上のコントラスト比（4.5:1以上）を確保
  • 色のみに依存しない情報提供
  • カラーユニバーサルデザインの採用
  • ハイコントラストモードでの表示確認
文字とフォント
  • 読みやすいフォントサイズ（16px以上）
  • 十分な行間・文字間隔の確保
  • 拡大表示（200%まで）への対応
  • 明朝体・ゴシック体の適切な使い分け
レイアウト
  • 論理的な情報構造の構築
  • 一貫したナビゲーション
  • レスポンシブデザインの採用
  • 横スクロールの回避

⌨️ 操作・ナビゲーション

キーボード操作
  • すべての機能へのキーボードアクセス
  • 論理的なタブ順序の設定
  • フォーカス表示の明確化
  • ショートカットキーの提供
ナビゲーション支援
  • パンくずリストの提供
  • サイトマップの設置
  • 検索機能の提供
  • ページ内リンクの設置
エラー防止・回復
  • 入力エラーの明確な表示
  • エラー修正方法の提示
  • フォーム入力の検証機能
  • 確認画面の提供

📄 コンテンツ・情報

テキスト
  • 平易で理解しやすい表現
  • 専門用語の解説提供
  • 読み仮名の併記（必要に応じて）
  • 適切な見出し構造の設定
画像・メディア
  • 画像への代替テキスト（alt属性）の設定
  • 情報画像と装飾画像の適切な区別
  • 動画への字幕・音声解説の提供
  • 音声コンテンツのテキスト版提供
文書・ファイル
  • PDF文書のアクセシブル化
  • ファイル形式・サイズの明示
  • 代替形式の提供（可能な場合）
  • ダウンロードファイルの説明

🛠️ 技術的対応

マークアップ
  • セマンティックなHTMLの使用
  • WAI-ARIAの適切な実装
  • HTML・CSSの文法チェック
  • ページタイトルの適切な設定
互換性
  • 主要ブラウザでの動作確認
  • スクリーンリーダーでの動作確認
  • モバイルデバイスでの動作確認
  • 古いブラウザでの基本機能保証

5. 主な対応機能

本サイトでは、以下のアクセシビリティ機能を提供しています。

⌨️

キーボードナビゲーション

マウスを使用できない方でも、キーボードのみですべての機能を利用できます。

主なキーボードショートカット
  • Tab：次の要素に移動
  • Shift + Tab：前の要素に移動
  • Enter：リンクの実行、ボタンの押下
  • Space：チェックボックスの切り替え
  • ↑↓：プルダウンメニューの選択
🔍

文字サイズ変更

ブラウザの機能を使用して、文字サイズを200%まで拡大できます。

文字サイズ変更方法
  • Windows： Ctrl + +（拡大）/ Ctrl + -（縮小）
  • Mac： ⌘ + +（拡大）/ ⌘ + -（縮小）
  • スマートフォン： ピンチイン・ピンチアウト
👁️

高コントラスト表示

文字と背景のコントラスト比を高く設定し、読みやすさを向上させています。

コントラスト比
  • 通常テキスト： 4.5:1以上
  • 大きなテキスト： 3:1以上
  • リンクテキスト： 3:1以上（下線付き）
🔊

スクリーンリーダー対応

視覚に障害のある方が使用するスクリーンリーダーで正しく読み上げられます。

対応スクリーンリーダー
  • NVDA（Windows）
  • PC-Talker（Windows）
  • VoiceOver（Mac・iOS）
  • TalkBack（Android）
📱

マルチデバイス対応

パソコン、タブレット、スマートフォンで同等の機能を利用できます。

対応デバイス
  • デスクトップPC（Windows・Mac・Linux）
  • タブレット（iPad・Android）
  • スマートフォン（iPhone・Android）
  • 音声ブラウザ・テキストブラウザ
🗺️

ナビゲーション支援

サイト内の移動を支援する各種機能を提供しています。

提供機能
  • サイトマップ：全ページの一覧
  • 検索機能：サイト内コンテンツ検索
  • パンくずリスト：現在位置の表示
  • ページ内リンク：長いページの目次機能

6. 検証・テスト

本サイトのアクセシビリティは、以下の方法で定期的に検証・テストを実施しています。

🔍 自動検証ツール

axe DevTools

ブラウザ拡張機能による自動チェック

  • WCAG準拠チェック
  • 色のコントラスト検証
  • フォーム要素の検証
WAVE (Web Accessibility Evaluation Tool)

WebAIMが提供するアクセシビリティ評価ツール

  • ページ全体の包括的チェック
  • エラー・警告の詳細表示

Lighthouse Accessibility Audit

Googleが提供するパフォーマンス・アクセシビリティ監査ツール

  • 総合的なアクセシビリティスコア
  • 具体的な改善提案
  • パフォーマンスとの統合評価

👤 手動テスト

キーボード操作テスト
  • マウスを使わない全機能テスト
  • タブ順序の論理性確認
  • フォーカス表示の視認性確認
  • ショートカットキーの動作確認
スクリーンリーダーテスト
  • NVDA・PC-Talkerでの読み上げ確認
  • 見出し構造の適切性確認
  • リンクテキストの理解しやすさ確認
  • フォーム要素のラベル確認
ユーザビリティテスト
  • 視覚障害者によるサイト利用テスト
  • 運動障害者による操作性テスト
  • 高齢者による使いやすさテスト
  • 認知障害者による理解しやすさテスト

📋 検証スケジュール

定期検証
  • 月次： 新規・更新ページの自動検証
  • 四半期： 主要ページの手動テスト
  • 年次： 全ページの包括的監査
随時検証
  • 新機能実装時の検証
  • デザイン変更時の検証
  • 利用者からの指摘時の検証

⚠️ 例外事項

以下の項目については、技術的制約や代替手段の提供により、一部準拠が困難な場合があります：

  • 外部埋め込みコンテンツ： YouTube動画等の第三者提供コンテンツ
  • リアルタイムコンテンツ： 緊急時の速報等、即時性を重視する情報
  • レガシーファイル： 古い形式で作成されたPDF等（代替手段を提供）
  • 第三者ツール： 外部サービスが提供するウィジェット等

これらについては、可能な限り代替手段を提供し、利用者への影響を最小限に抑えます。
